1. Структурура файла Data PSD v.2.0.xlsx   - там "сигнатура" бинарного файла типа data_psd_2025_07_18__16_47_59.bin
2. Структура файла Waveform PSD v.3.0.xlsx - там "сигнатура" бинарного файла типа waveform_psd_2025_07_18__16_47_59.bin

3. data_psd_2025_07_18__16_47_59.bin     - сюда записаны легкие данные - заряды, времена, ... 
4. waveform_psd_2025_07_18__16_47_59.bin - сюда записаны сырые данные - формы сигналов 

5. test.cpp - скрипт, который дипсик написал для преобразования файлов "data_psd_..." (легких данных) в ROOT-дерево
6. prob.cpp - скрипт для анализа данных из ROOT-дерева
7. psdData_3.root - пример ROOT-дерева по данным "data_psd_..." (легким данным)


Необходимо:
А) Проверить и, возможно, скорректировать скрипт test.cpp для "легких" данных
Б) По образу и подобию написать скрипт для "сырых" данных (вейвформ)

Можно объединить два скрипта в один. Тогда на выходе будет записано ROOT-дерево, содержащее элементы нового класса "Event", 
в которое будут входить:
- вейформа сигнала
- амплитуда сигнала
- заряд 1
- заряд 2
- временные метки всякие
- прочее, что можно достать из этих бинарников
